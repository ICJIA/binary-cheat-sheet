Introduction: Why Binary Matters

This tutorial provides a clear, visual introduction to binary numbers and their significance in computing.

1. Introduction to Binary Numbers

Understanding Number Systems and the Concept of Base

Our everyday decimal system is base-10, using ten digits (0-9). The position of a digit determines its value based on powers of 10 (e.g., 325 = 3×102 + 2×101 + 5×100). The "base" (or radix) defines how many unique symbols are used and the power associated with each position.

Binary is base-2, using only two digits (0 and 1). Place values are powers of 2 (..., 23, 22, 21, 20). Why base 2? As mentioned, it directly matches the "on/off" physical states of computer circuits, making hardware design simpler and more robust than trying to reliably represent ten different voltage levels for a base-10 system.

Throughout this guide, we'll include code examples in JavaScript, Python, and Z80 Assembly. The Z80 microprocessor (1976) was chosen for its clear, educational instruction set and historical significance. While modern processors are more complex, the Z80's assembly language provides an excellent foundation for understanding how computers work at the lowest level.

Why Z80 Assembly?

The Z80 microprocessor, developed by Zilog in 1976, is an 8-bit CPU that was widely used in early personal computers, game consoles, and industrial applications. While it's an older architecture, it's particularly well-suited for educational purposes because:

  • Its instruction set is clean and logical, making it easier to understand fundamental computing concepts
  • It has a manageable number of registers (A, B, C, D, E, H, L) and simple addressing modes
  • It uses 8-bit operations which are perfect for learning binary concepts
  • Many early computers (like the ZX Spectrum, TRS-80, and MSX) used the Z80, making it historically significant

The Altair 8800 Connection

The Altair 8800, released in 1975, was one of the first commercially successful personal computers. While it used the Intel 8080 processor (which the Z80 was designed to be compatible with), it's included in our examples because:

  • It provides a clear example of how computers work at the binary level
  • Its front-panel switches and LEDs make binary concepts tangible
  • It represents a crucial point in computing history where programming was done directly in machine language
  • It helps bridge the gap between abstract binary concepts and real-world computer architecture

By studying these older architectures, we gain a deeper understanding of fundamental computing concepts that remain relevant today, even in modern 64-bit processors. The principles of binary arithmetic, memory addressing, and CPU operations are universal, and learning them through simpler architectures makes the concepts more accessible.

Relating Binary to Decimal Numbers

Counting works similarly, but you "run out" of digits much faster:

  • Decimal 0 = Binary 0
  • Decimal 1 = Binary 1
  • Decimal 2 = Binary 10 (ran out of digits, carry over the 1)
  • Decimal 3 = Binary 11
  • Decimal 4 = Binary 100 (carry over again)

Decimal to Binary Representation

To represent a decimal number in binary, find the combination of powers of 2 that sum up to that number. For decimal 5:

  1. Largest power of 2 ≤ 5 is 4 (22). Need a '1' in the 22 place. Remainder = 1.
  2. Next power is 2 (21). 2 > 1. Need a '0' in the 21 place. Remainder = 1.
  3. Next power is 1 (20). 1 ≤ 1. Need a '1' in the 20 place. Remainder = 0.

Result: Decimal 5 = Binary 101 (1*4 + 0*2 + 1*1 = 5).

Decimal 5 = Binary 101

Binary Digit 1 0 1
Power of 2 22 21 20
Decimal Value 4 2 1
Contribution 1×4=4 0×2=0 1×1=1
Total: 4 + 0 + 1 = 5

2. Digits: Bits and Their Place Value

Defining "Bit" and Its Significance

A "bit" (short for binary digit) is the most basic unit of information in computing, representing a single 0 or 1. Why group them? A single bit doesn't convey much information (only two possibilities). By grouping bits, we can represent more complex data. A common grouping is a "byte," which consists of 8 bits.

Why 8 bits in a byte? This became a de facto standard largely due to early hardware like the popular Intel 8080 processor (used in the Altair 8800) which had an 8-bit architecture. An 8-bit byte can represent 28 = 256 different values (0 to 255), which was sufficient to represent characters (like letters A-Z, numbers 0-9, punctuation via ASCII encoding), basic instructions, and other small data units efficiently.

Bytes (and multiples like Kilobytes (KB), Megabytes (MB), Gigabytes (GB)) are the standard units for measuring file sizes, memory capacity, and data transfer rates.

Place Values in Binary (Powers of 2)

Just like decimal, binary uses place value, but with powers of 2. The rightmost bit is the 20 (1s) place, the next is 21 (2s), then 22 (4s), and so on.

Binary Place Values

... Position 4 (24) Position 3 (23) Position 2 (22) Position 1 (21) Position 0 (20) (LSB)
Value ... 16 8 4 2 1

Bit Contribution Example

To find the decimal value of a binary number, sum the place values where a '1' appears. For binary 10110:

Converting 10110

Bit 1 0 1 1 0
Place Value 16 8 4 2 1
Contribution 1×16=16 0×8=0 1×4=4 1×2=2 0×1=0
Total: 16 + 0 + 4 + 2 + 0 = 22

Code Example: Checking a Specific Bit

Why check specific bits? Often used in low-level programming, hardware interaction, or when using bit flags (where each bit in a byte/integer represents a specific boolean option or status). This example checks if the bit representing '2' is set. 

// JavaScript: Check if the 2nd bit (position 1, value 2^1=2) is set
let number = 5; // Binary: 0101
let bitPosition = 1; // Check the bit at index 1 (0-indexed from right)

// Create a mask: Shift 1 left by 'bitPosition' places (1 << 1 -> 0010)
let mask = 1 << bitPosition;

// Use bitwise AND. If the result is non-zero, the bit was set.
let isSet = (number & mask) !== 0;

console.log(`Number: ${number} (${number.toString(2).padStart(4, '0')})`);
console.log(`Is bit at pos ${bitPosition} set? ${isSet}`); // Output: false

number = 7; // Binary: 0111

console.log(`\nNumber: ${number} (${number.toString(2).padStart(4, '0')})`);
console.log(`Is bit at pos ${bitPosition} set? ${isSet}`); // Output: true

3. Bit Significance and Byte Order: LSB, MSB, and Endianness

LSB and MSB

Within a binary number, bits have different significance:

  • LSB (Least Significant Bit): The rightmost bit (20 place). Changing it has the smallest effect on the number's value.
  • MSB (Most Significant Bit): The leftmost bit (highest power of 2 place). Changing it has the largest effect. In signed numbers, the MSB often indicates the sign (0=positive, 1=negative in Two's Complement).

Byte 10110101:

MSB --> 1 0 1 1 0 1 0 1 <-- LSB

Endianness: Why Does Byte Order Matter?

When numbers (like a 32-bit integer) occupy multiple bytes in memory, the processor needs a consistent way to order those bytes. This order is endianness. Why does it exist? It stems from different historical CPU design choices. There's no single "correct" way, just different conventions:

  • Little-Endian: Stores the Least Significant Byte (LSB) at the lowest memory address. Think "Little end first". This is used by Intel/AMD x86/x64 processors. Some arithmetic operations (like addition on multi-byte numbers) can sometimes be implemented slightly more easily this way.
  • Big-Endian: Stores the Most Significant Byte (MSB) at the lowest memory address. Think "Big end first". This is often called "network byte order" because standard internet protocols (like IP, TCP, UDP) mandate big-endian. It can feel more natural to read for humans, similar to how we write numbers. Used by older architectures (Motorola 68k, SPARC) and some modern ones (like IBM mainframes).

Why care? If you read binary data (like from a file or network socket) created on a system with one endianness and try to interpret it on a system with different endianness without conversion, you'll get incorrect values! This is a common source of bugs in networking and cross-platform development.

4. The Power of Manipulation: Bitwise Operations

Introduction

Bitwise operators work directly on the binary representations of numbers. Why use them? They are extremely fast (often single CPU instructions) and essential for low-level tasks like hardware control, data compression, encryption algorithms, graphics rendering, and performance optimization. They allow precise control over individual bits.

Detailed Explanations and Examples

AND (&)

Result bit is 1 only if both input bits are 1. Use Cases: Often used for "masking" – selecting specific bits (by ANDing with a mask where only desired bits are 1) or checking if a specific bit (a "flag") is set. 

  0101 (5)  // Input A
& 0011 (3)  // Input B (Mask)
------
  0001 (1)  // Result
let a = 5;
let b = 3;
console.log(a & b); // Output: 1
a = 5
b = 3
print(a & b) # Output: 1
; Z80 Assembly: AND operation
; AND performs bitwise AND between A and operand
; Result is stored in A

    ; Load values into registers
    LD A, 5       ; A = 00000101
    LD B, 3       ; B = 00000011

    ; Perform AND operation
    AND B         ; A = A AND B
    ; Result: A = 00000001 (1)

    ; Common uses:
    ; 1. Masking bits (AND with 1 keeps bit, AND with 0 clears bit)
    LD A, 0xFF    ; A = 11111111
    AND 0x0F      ; Keep only lower 4 bits: A = 00001111

    ; 2. Testing if bit is set
    LD A, 0x42    ; A = 01000010
    AND 0x02      ; Test bit 1
    JR NZ, bit_set ; Jump if bit was set

OR (|)

Result bit is 1 if at least one input bit is 1. Use Cases: Commonly used to "set" specific bits or flags to 1 (by ORing with a mask where the desired bits are 1). 

  0101 (5)  // Input A
| 0011 (3)  // Input B (Mask)
------
  0111 (7)  // Result
let a = 5;
let b = 3;
console.log(a | b); // Output: 7
a = 5
b = 3
print(a | b) # Output: 7
; Z80 Assembly: OR operation
; OR performs bitwise OR between A and operand
; Result is stored in A

    ; Load values into registers
    LD A, 5       ; A = 00000101
    LD B, 3       ; B = 00000011

    ; Perform OR operation
    OR B          ; A = A OR B
    ; Result: A = 00000111 (7)

    ; Common uses:
    ; 1. Setting bits (OR with 1 sets bit, OR with 0 keeps bit)
    LD A, 0x42    ; A = 01000010
    OR 0x01       ; Set bit 0: A = 01000011

    ; 2. Combining flags
    LD A, 0x01    ; Flag 1 set
    OR 0x02       ; Set flag 2: A = 0x03

NOT (~)

Inverts every bit (0→1, 1→0). Why the weird result (-x - 1)? It operates on the full integer representation (e.g., 32 bits) using Two's Complement for negative numbers. It's less commonly used directly than other operators but is part of boolean algebra principles. 

~ 0101 (5) = ...11111010 (-6 in Two's Complement)

Note: ~x is equivalent to -x - 1. 

let a = 5;
console.log(~a); // Output: -6
a = 5
print(~a) # Output: -6
; Z80 Assembly: NOT operation
; CPL performs bitwise NOT on accumulator A
; Result is stored in A

    ; Load value into accumulator
    LD A, 5       ; A = 00000101

    ; Perform NOT operation
    CPL           ; A = 11111010 (-6 in Two's Complement)
    ; Note: CPL is equivalent to XOR 0xFF

    ; Common uses:
    ; 1. Inverting a mask
    LD A, 0x0F    ; A = 00001111 (mask for lower 4 bits)
    CPL           ; A = 11110000 (mask for upper 4 bits)

    ; 2. Toggling all bits
    LD A, 0x42    ; A = 01000010
    CPL           ; A = 10111101

XOR (^)

Result bit is 1 if input bits are different. Use Cases: Toggling specific bits (XORing with a mask flips the bits where the mask is 1), simple checksums/hashing, basic encryption (XORing data with a key), swapping values without a temporary variable. 

  0101 (5)  // Input A
^ 0011 (3)  // Input B (Mask)
------
  0110 (6)  // Result (Bits at pos 0 and 2 were different)
let a = 5; // 0101
let b = 3; // 0011
console.log(a ^ b); // Output: 6 (0110)
a = 5 # 0b0101
b = 3 # 0b0011
print(a ^ b) # Output: 6 (0b0110)
; Z80 Assembly: XOR operation
; XOR performs bitwise XOR between A and operand
; Result is stored in A

    ; Load values into registers
    LD A, 5       ; A = 00000101
    LD B, 3       ; B = 00000011

    ; Perform XOR operation
    XOR B         ; A = A XOR B
    ; Result: A = 00000110 (6)

    ; Common uses:
    ; 1. Toggling specific bits
    LD A, 0x42    ; A = 01000010
    XOR 0x02      ; Toggle bit 1: A = 01000000

    ; 2. Simple encryption/decryption
    LD A, 0x55    ; Data to encrypt
    XOR 0x33      ; XOR with key: A = 0x66
    XOR 0x33      ; XOR again to decrypt: A = 0x55

Left Shift (<<)

Shifts all bits to the left by N positions, filling the rightmost N positions with 0s. Why use it? It's a very fast way to multiply by powers of 2 (x << N is equivalent to x * 2N). Also used for packing data efficiently into smaller spaces. 

  0011 (3) << 2 = 1100 (12)  (3 * 2^2 = 3 * 4 = 12)
let a = 3;
let n = 2;
console.log(a << n); // Output: 12
a = 3
n = 2
print(a << n) # Output: 12
; Z80 Assembly: Left Shift
; SLA performs arithmetic left shift (multiply by 2)
; SLL performs logical left shift (multiply by 2, sets bit 0)
; Result is stored in A or specified register

    ; Load value into register
    LD A, 3       ; A = 00000011

    ; Perform left shift (multiply by 2)
    SLA A         ; A = 00000110 (6)
    ; Note: SLA preserves sign bit, shifts left, sets bit 0 to 0

    ; Common uses:
    ; 1. Fast multiplication by powers of 2
    LD A, 5       ; A = 00000101
    SLA A         ; Multiply by 2: A = 00001010 (10)
    SLA A         ; Multiply by 4: A = 00010100 (20)

    ; 2. Packing bits (using SLL)
    LD A, 0x42    ; A = 01000010
    SLL A         ; Shift left, set bit 0: A = 10000101

Right Shift (>>, >>>)

Shifts all bits to the right by N positions. Bits shifted off the right end are discarded. Why use it? It's a very fast way to perform integer division by powers of 2 (x >> N is roughly x / 2N, truncating the result). Also used for unpacking data or isolating specific parts of a number.

  • >> (Sign-propagating/Arithmetic): Fills vacated left bits with the original sign bit. Preserves the number's sign. This is the standard behavior in Python and for signed integers in JS/Java/C++.
  • >>> (Zero-fill/Logical - JS only): Always fills vacated left bits with 0. Treats the number as unsigned for the shift operation. 
  0110 (6) >> 1 = 0011 (3)   (6 / 2^1 = 3)
  -6 >> 1 = -3 (Sign preserved)
let a = 6;
let n = 1;
console.log(a >> n);  // Output: 3
console.log(a >>> n); // Output: 3 (Same for positive numbers)

let negA = -6;
console.log(negA >> n);  // Output: -3 (Sign-propagating)
console.log(negA >>> n); // Output: 2147483645 (Zero-fill)
# Python's >> is always sign-propagating (arithmetic)
a = 6
n = 1
print(a >> n) # Output: 3

neg_a = -6
print(neg_a >> n) # Output: -3

# Logical right shift requires masking if needed
width = 32
mask = (1 << width) - 1
print(f"Logical approx: {(neg_a & mask) >> n}") # Output: 2147483645
; Z80 Assembly: Right Shift
; SRA performs arithmetic right shift (divide by 2, preserves sign)
; SRL performs logical right shift (divide by 2, sets bit 7 to 0)
; Result is stored in A or specified register

    ; Load value into register
    LD A, 6       ; A = 00000110

    ; Perform right shift (divide by 2)
    SRA A         ; A = 00000011 (3)
    ; Note: SRA preserves sign bit, shifts right, sets bit 7 to sign bit

    ; Common uses:
    ; 1. Fast division by powers of 2
    LD A, 20      ; A = 00010100
    SRA A         ; Divide by 2: A = 00001010 (10)
    SRA A         ; Divide by 4: A = 00000101 (5)

    ; 2. Extracting bits (using SRL)
    LD A, 0x42    ; A = 01000010
    SRL A         ; Shift right, clear bit 7: A = 00100001

5. Bridging the Gap: Converting Binary to Decimal

Why convert to decimal? While computers work in binary, humans primarily understand and communicate using the decimal system. Converting binary values to decimal allows us to interpret computer data, memory contents, or calculation results in a familiar format.

Positional Value Method

This method directly applies the definition of base-2 place values. Sum the decimal values (powers of 2) corresponding to each '1' bit in the binary number. Example: 10110 → 1*16 + 0*8 + 1*4 + 1*2 + 0*1 = 22.

Doubling Method

An alternative algorithm often easier for mental calculation. Start with 0, scan the binary number from left to right. For each bit, double your current total and add the bit's value (0 or 1). Example: 10110 → (((((0*2)+1)*2)+0)*2+1)*2+1)*2+0 = 22.

Fractional Parts

For bits after the binary point, use negative powers of 2 (1/2, 1/4, 1/8...). Example: 10.11 → (1*2 + 0*1) + (1*0.5 + 1*0.25) = 2 + 0.75 = 2.75.

Code Example: Binary String to Decimal

Most languages provide built-in functions, but understanding the manual methods (like the positional one shown here) clarifies the concept. 

// 1. Built-in
let binStr = "10110";
console.log(parseInt(binStr, 2)); // Output: 22

// 2. Manual Positional
function binToDec(binStr) {
  let dec = 0;
  let power = 0;
  for (let i = binStr.length - 1; i >= 0; i--) {
    if (binStr[i] === '1') {
        dec += Math.pow(2, power);
    }
    power++;
  }
  return dec;
}

function binFracToDec(binStr) {
    const [intP, fracP=""] = binStr.split('.');
    let dec = parseInt(intP, 2) || 0;
    for (let i = 0; i < fracP.length; i++) {

            dec += Math.pow(2, -(i + 1));
        }
    }
    return dec;
}

def binary_to_decimal(bin_str):
  dec = 0

    power += 1

def binary_fraction_to_decimal(bin_str):
    int_part, _, frac_part = bin_str.partition('.')
    dec = int(int_part, 2) if int_part else 0
    frac_val = sum(int(bit) * 2**-(i + 1) for i, bit in enumerate(frac_part))
    return dec + frac_val

6. Hexadecimal Harmony: Converting Binary to Hexadecimal

Hexadecimal (base-16) uses digits 0-9 and letters A-F. Why use Hex? Binary numbers get very long very quickly (e.g., a 32-bit number is 32 0s/1s!). Hexadecimal provides a much more compact way to represent binary data. Since 16 is a power of 2 (16 = 24), exactly four binary digits (bits) map perfectly to one hexadecimal digit. This makes conversion straightforward and is much easier for humans to read and write than long binary strings.

Use Cases: Hex is extremely common in computing for representing memory addresses, defining colors (e.g., CSS color codes like `#FF0000` for red), displaying raw data in memory dumps or file hex editors, and in assembly language programming.

Grouping

The key is grouping the binary string into sets of four bits, starting from the right (or binary point). Pad the leftmost group with leading zeros if needed. Example: 1101011010 → 0011 0101 1010

Conversion Table (4-bit to Hex)

0000=0 0001=1 0010=2 0011=3
0100=4 0101=5 0110=6 0111=7
1000=8 1001=9 1010=A 1011=B
1100=C 1101=D 1110=E 1111=F

Example

Convert each 4-bit group: 0011→3, 0101→5, 1010→A. Combine them: 35A16.

Code Example: Binary String to Hexadecimal

Using built-ins (often easiest) and the manual grouping method.

// 2. Manual Grouping
function binToHex(binStr) {
  // Pad with leading zeros to make length a multiple of 4
  while (binStr.length % 4 !== 0) {
    binStr = '0' + binStr;
  }

  let hex = "";
  const binToHexMap = { // Complete map
    "0000": "0", "0001": "1", "0010": "2", "0011": "3",
    "0100": "4", "0101": "5", "0110": "6", "0111": "7",
    "1000": "8", "1001": "9", "1010": "A", "1011": "B",
    "1100": "C", "1101": "D", "1110": "E", "1111": "F"
  };

  for (let i = 0; i < binStr.length; i += 4) {
    const chunk = binStr.substring(i, i + 4);
    hex += binToHexMap[chunk]; // Use map
  }
  return hex;
}

7. Octal Insights: Converting Binary to Octal

Octal (base-8) uses digits 0-7. Why use Octal? Similar to hex, it's more compact than binary. Since 8 = 23, one octal digit maps perfectly to three binary digits. While less common than hex today, octal was historically significant, especially in older computer systems (like DEC PDP series) and is still used in some contexts, most notably for representing file permissions in Unix/Linux systems (e.g., `chmod 755`).

Grouping

Group the binary string into sets of three bits, starting from the right. Pad the left with zeros if needed. Example: 1101011010 → 001 101 011 010

Conversion Table (3-bit to Octal)

000=0 001=1 010=2 011=3
100=4 101=5 110=6 111=7

Example

Convert each 3-bit group: 001→1, 101→5, 011→3, 010→2. Combine them: 15328.

  };

  if (oct.length > 1 && oct.startsWith('0')) {

  }
  return oct;
}

8. Quick Translations: Tips and Tricks

These tips help solidify understanding and speed up conversions:

  • Grouping (Why?): Grouping by 3 (Octal) or 4 (Hex) works because 8 and 16 are powers of 2. It's a shortcut based on the mathematical relationship between the bases, avoiding direct decimal conversion.
  • Memorize Small Patterns: Knowing 0-7 (for Octal) and 0-F (for Hex) binary equivalents makes the grouping method much faster.
  • Powers of 2 are Key: Always remember the place values (1, 2, 4, 8, 16...) for understanding binary magnitude and for positional conversion to decimal.
  • Positional Value (Bin→Dec): This is the most fundamental method for understanding how a binary number represents its decimal value.
  • Check Work: Use programming language functions or online calculators to verify your manual conversions, especially while learning.

9. Historical Context: Binary Input and the Altair 8800

The MITS Altair 8800 (1975) is a landmark machine that helped launch the personal computer era. For me personally, this was the first computer I ever used around 1981, and it sparked my lifelong interest in computing. Why is it relevant here? It vividly illustrates the fundamental role of binary. Unlike modern computers with layers of abstraction (keyboards, operating systems, graphical interfaces), programming the first Altairs involved interacting directly with the machine's binary language.

Historical Background

The Altair 8800 was introduced in 1975 by Micro Instrumentation and Telemetry Systems (MITS) as a mail-order kit for $439. It was based on the Intel 8080 microprocessor and became the first commercially successful personal computer. The machine was featured on the cover of Popular Electronics in January 1975, which caught the attention of Paul Allen and Bill Gates.

In 1975, Bill Gates and Paul Allen developed Altair BASIC, which became Microsoft's first product. This was a significant milestone as it made the Altair more accessible to users who weren't comfortable with machine language programming. The first version of Altair BASIC was written on a PDP-10 minicomputer at Harvard University, where Gates was a student.

Binary Switches for Input

Users had to manually set rows of toggle switches on the front panel to represent binary 1s (switch up) and 0s (switch down). They would set the switches for a memory address, deposit that address, then set the switches for the binary instruction or data byte, and deposit that value into memory. Output was read from blinking LEDs, also representing binary states.

Example: Simple Program Input

Here's what it would look like to input a simple program that adds two numbers. Each instruction had to be entered in binary, one byte at a time:

Peripherals and Expansion

While the Altair 8800 initially had no keyboard or display, MITS later offered expansion options:

  • Keyboard: The MITS 88-KBD keyboard was released in 1976, connecting through the S-100 bus
  • Display: The MITS 88-VDT terminal was an optional video display unit
  • Storage: The MITS 88-ACR cassette interface allowed saving programs to audio cassettes
  • Memory: The base model came with 256 bytes of RAM, expandable to 64KB

Legacy and Impact

The Altair 8800's influence extends far beyond its initial release. It inspired the formation of the Homebrew Computer Club, where Steve Wozniak and Steve Jobs first met. The machine's S-100 bus became an industry standard, and its architecture influenced many early personal computers. The development of Altair BASIC marked the beginning of Microsoft, which would go on to become one of the world's largest technology companies.

For many early computer enthusiasts, including myself, the Altair 8800 was their first introduction to computing. The experience of programming in binary using toggle switches provided a deep understanding of how computers work at their most fundamental level - something that's often abstracted away in modern computing.
